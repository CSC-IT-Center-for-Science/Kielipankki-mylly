Drop/count selected attributes

The "drop with count" operation projects an input relation to other than the selected attributes with the addition of a new count attribute to record the multiplicity (in the input) relation of the kept combinations.

The input and output relations are represented as TSV files.

Parameters

  • name for the count attribute
  • zero or more attributes

Attributes are presented as menus that list the attributes that the relation has. EMPTY indicates that the parameter is not used.

Selecting the same attribute more than once is harmless.

(Selecting all attributes produces a relation with no attributes. There are two such relations.)

Example

The input relation consists of word forms annotated with sentence and token numbers and lemmas.

word tok sen lemma
on 13olla
se 23se
niin33niin
on 15olla
ei 14ei
ole 24olla

The count attribute is named count. (The default name is cMcount.)

When the sentence number and token number are dropped, the two occurrences of olla as on melt into one record.

count lemma word
1ei ei
1se se
1niinniin
2ollaon
1ollaole

When the sentence number, the token number, and the word form itself are dropped, all three occurrences of olla melt into one record.

count lemma
1ei
1se
3olla
1niin
